Week 8
Sorting (merge sort, quick sort, bubble sort)
April 17th, 2015
Merge Sort: comparison-based sorting.

Divide the unsorted list into X sublists, each containing 1 element. Now repeatedly merge sublists to produce new sorted sublists until there is only 1 sublist remaining. This will be the sorted list.

Pro: Best choice for a merged link list. Default sorting algorithm for Perl, Java, Python.

Quick Sort: divide and conquer sorting.

Divide the unsorted list into two smaller arrays: the lowest # elements and highest # elements. Based off of picking a 'pivot' number, and sort above or below that number to create the 2 arrays. Continue separating until there is one sorted list.

Pros: Algorithm of choice in many situations since its easy to implement and uses less memory during execution.

Cons: It is recursive.

Bubble Sort: comparison-based sorting.

Compare two elements at the same time and loop through the array only comparing 2 at a time. No dividing into arrays.

Named for the smaller elements that will bubble to the top through the sort.

Pros: The ability to check and see if any swaps are made in a pass, if not, then we know that the array is sorted and we can stop the sort.

Cons: Too small and simple for most problems
