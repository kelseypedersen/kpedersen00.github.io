Variable Scope
April 5th, 2015

What kinds of variables does Ruby support? Where are they accessible? When would you use them and what makes them different?

What is variable scope?

Defines in a program where a variable is accessible. It's declared by using a special character at the start of a variable name as outlined in the table below

Name Begins With === Variable Scope

$ <===> Global Variable
@ <===> Instance Variable
[a-z] or _ <===> Local Variable
[A-Z] <===> Constant
@@ <===> Class Variable

Determining the variable scope

It's usually fairly easy to understand the variable's scope based on the symbol at the front of the variable. However, sometimes you may need to figure out the variable scope programmatically. To do so, you use the "defined?" method.

x = 100
defined? x
=> "local-variable"

@x = 100
defined? x
=> "instance-variable"

Global Variables ($)

Global Variables are used to create a variable that is accessible anywhere in the Ruby program. The use of global variables is NOT encouraged because since they are visible anywhere in the entire program, they can also be changed anywhere in the program - making debugging more difficult.

Instance Variables (@)

Instance values are local to specific instances of the class. Other objects of the same class have their own local values, which are independent of changes made in other objects.

Local Variables ([a-z])

Local Variables are only used inside the method or loop where it's defined.

Constant ([A-Z])

Constants should not be changed after assigned a value. In Ruby, they can be changed but it is not best practice. Constants declared in a class will only be accessible inside the class. Constants declared outside a class are assigned global scope.

Class Variable (@@)

Class Variables are shared across all instances of the class. There is only one variable value for all objects created from the class. This means that if the value changes in one class, the value changes for all the other object instances.
